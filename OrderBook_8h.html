Detailed Description

Defines the core matching engine logic.

The OrderBook maintains the state of all active Buy and Sell orders. It uses a hybrid data structure approach:

  1. Price Tress (std::map): Keep orders sorted by Price.
  • Bids: Sorted Descending (Highest price first).
  • Asks: Sorted Ascending (Lowest price first).
  1. Order Map (std::unordered_map): Maps OrderId -> Order* for O(1) lookups/cancellations.
  2. Memory Pool: All order objects are allocated from a pre-allocated slab to avoid heap fragmentation.

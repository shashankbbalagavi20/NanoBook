Detailed Description

Defines a pre-allocated memory pool (Slab Allocator) for fast object management.

Calling 'new' or 'malloc' is avoided because it triggers a system call (context switch) and non-deterministic heap fragmentation. This ObjectPool pre-allocates a contiguos block of memory (std::vector)

  • Allocation: O(1) - Pops a free index from the stack.
  • Deallocation: O(1) - Pushes the index back onto the stack.
  • Cache Friendliness: LIFO (Last-In, First-Out) strategy ensures we reuse the most recently released object(hot cache).

Detailed Description

A thread-safe wrapper (Decorator) for the standard OrderBook.

This class applies the "Monitor" synchronization pattern:

  1. It holds an instance of OrderBook.
  2. It owns a SpinLock.
  3. It exposes the same interface as OrderBook, but wraps every method call in a lock()/unlock() block. This ensures that multiple threads (e.g., Worker Threads) can safely submit orders simultaneously without causing race conditions or memory corruption.
